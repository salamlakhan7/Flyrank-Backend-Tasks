# FL-02 — Frame It as Cases: Work That Speaks for Itself

A framed case study built around the actual FlyRank Backend and AI
Engineering internship work (BE-01 through BE-06), plus a voice card
and one before/after edit showing generic AI phrasing cut down to
something specific and true.

## What's in the PDF

1. **Voice card** — five to seven words for how I want to sound:
   *"Direct, honest about gaps, technical, no fluff."*
2. **The case** — three beats: the problem (environment and git
   friction, not the code itself), what I did and decided (fixing a
   nested `.git` submodule issue, switching HTTPS to SSH after
   repeated `408` timeouts, moving off Play with Docker to Codespaces
   once it was deprecated, building a repository pattern in BE-04, and
   idempotency/retries/alerts in BE-06), and what came of it (one
   public repo, four real assignments deep, each with working code and
   real proof it runs).
3. **Bio and contact/CTA** — written in the same voice, pointing at
   the actual GitHub repo.
4. **Before/after** — one generic AI-sounding line versus the edited
   version, showing what got cut and why.

## Why this case, specifically

The brief asks to start with what's actually being made in the
internship. BE-01 through BE-06 is real, ongoing work with genuine
friction in it (git conflicts, a deprecated tool, real debugging
sessions), which is what makes it usable as a case rather than a
polished-after-the-fact summary.
